# 跨项目逆向重构工作流（proto-refactor-pipeline）

基于 Claude Code Skill 实现的自动化工作流：将 AI 快速原型逆向重构为生产级代码，支持任意目标项目。

---

## 解决什么问题

产品团队用 AI 快速搭建平台（妙悟、Lovable、V0 等）生成的原型代码，无法直接用于生产：
- 单文件 1000+ 行，结构混乱
- 不符合团队编码规范和 ESLint 规则
- 权限逻辑散落各处，无法维护
- 手动迁移容易遗漏功能  

本工作流自动完成：**读原型 → 提需求 → 按规范重写 → 验证零遗漏**

> 工作流保证不漏业务逻辑，样式和交互细节有些需要自己调整

---

## 核心能力

| 能力 | 说明 |
|------|------|
| 逆向提取需求 | 从原型代码自动生成结构化需求文档 |
| 规范化重写 | 按目标项目的技术栈和编码规范重新实现 |
| 功能回测 | 逐项验证原型功能 100% 覆盖，无遗漏 |
| 增量同步 | 产品出新包时只同步变化部分，不影响已有代码 |
| 断点恢复 | 中断后说「继续」即可接着跑 |
| Token 优化 | 内置 TokenGuard 策略，节省约 70% token 消耗 |
| 跨项目通用 | 首次使用自动读取目标项目规范，无需手动配置 |

## 工作流程

```
原型代码包
    │
    ▼
┌─────────────────────────────────────────────┐
│  Phase 1 · Migrator（迁移）                   │
│  扫描原型 → 生成需求文档 → 按规范逐组件实现    │
├─────────────────────────────────────────────┤
│  Phase 2 · Refiner（完善）                    │
│  深度组件化拆分 → hook 抽取 → 权限收敛         │
├─────────────────────────────────────────────┤
│  Phase 3 · Cleaner（清理）                    │
│  ESLint 合规 → 删除重复/无用代码               │
├─────────────────────────────────────────────┤
│  Phase 4 · Verifier（回测）                   │
│  逐项比对功能覆盖率 → 自动补全缺失项           │
└─────────────────────────────────────────────┘
    │
    ▼
生产级代码（100% 功能覆盖）
```

## 安装

**一键安装：**

```bash
curl -fsSL https://raw.githubusercontent.com/WeiSelfdiscipline/proto-refactor-pipeline/main/install.sh | bash
```

或手动复制：

```bash
git clone https://github.com/WeiSelfdiscipline/proto-refactor-pipeline.git ~/.claude/skills/proto-refactor-pipeline
```

目录结构：

```
proto-refactor-pipeline/
├── SKILL.md                  # 入口：触发条件 + 执行协议
└── references/
    ├── token-guard.md        # TokenGuard 策略（省 token）
    ├── migrator.md           # Phase 1 迁移规范
    ├── refiner.md            # Phase 2 完善规范
    ├── cleaner.md            # Phase 3 清理规范
    ├── verifier.md           # Phase 4 回测规范
    └── zlj-spec.md           # 示例项目规范（参考用）
```

## 使用教程

### 第一步：在 Claude Code 中打开目标项目

用 Claude Code 打开你要接收重构代码的目标项目目录（不是原型目录）。

### 第二步：触发 Skill

在对话框输入触发词 + 原型代码路径：

```
重构到目标项目 /path/to/prototype
```

> 原型路径是产品给你的 AI 生成代码包的目录，目标项目是你们团队的生产代码仓库。

### 第三步：首次配置（仅一次）

首次使用时，Skill 会自动引导配置，全程问答式，无需手动编辑任何文件：

1. **询问基础路径** — 目标组件目录、服务层文件路径
2. **自动读取规范** — 从目标项目读取 `.eslintrc`、`CLAUDE.md`、`package.json`、目录结构
3. **生成项目规范** — 整合为 `.agent/project_spec.md`
4. **询问补充约束** — 可以追加命名规则、禁用写法等团队约定
5. **保存配置** — 写入 `.agent/project_config.json`，后续不再询问

配置保存后，下次直接从第二步开始，跳过本步骤。

### 第四步：等待执行

Skill 自动按顺序跑完四个阶段，每个阶段结束输出简报。全程无需干预。

中途被打断没关系，下次说「继续」即可从断点接着跑。

### 常见场景

**产品出了新原型包：**
```
新包来了 /path/to/new-prototype
```
只同步变化部分，不动已有代码。

**只想看看原型有哪些功能：**
```
生成需求文档 /path/to/prototype
```
只分析不写代码，输出结构化需求文档。

**只想跑功能覆盖率验证：**
```
回测功能
```

## 触发词速查

| 说什么 | 做什么 |
|--------|--------|
| 「重构到目标项目」+ 路径 | 全流程：迁移→完善→清理→回测 |
| 「新包来了」+ 路径 | 增量同步：对比差异→只改变化部分→回测 |
| 「原型更新了」/ 「增量同步」 | 增量同步（同一仓库 git pull 后，自动用上次路径） |
| 「生成需求文档」+ 路径 | 只读代码生成需求文档，不写代码 |
| 「完善代码」 | 只跑组件化拆分 |
| 「清理代码」 | 只跑 ESLint 合规 + 删无用代码 |
| 「回测功能」 | 只跑功能覆盖率验证 |
| 「继续」 | 从上次断点恢复 |

## 产出物

运行后在原型目录下生成 `.agent/`：

```
{原型路径}/.agent/
├── project_config.json     # 项目配置（首次生成）
├── project_spec.md         # 自动生成的项目规范
├── status.json             # 实时进度（支持断点恢复）
├── codebase_map.json       # 原型功能全景图
├── task_list.json          # 任务列表
├── coverage_report.json    # 功能覆盖率报告
└── 需求文档.md             # 逆向提取的业务需求文档
```

---

## 注意事项

- **防覆盖保护**：目标目录已有代码时，自动建议走增量/回测模式
- **Token 优化**：大文件分片读取，相比全量读取节省约 70%
- **断点续传**：中断后说「继续」即可恢复，无需从头执行
- **跨项目**：所有路径从配置文件读取，不绑定任何特定项目
